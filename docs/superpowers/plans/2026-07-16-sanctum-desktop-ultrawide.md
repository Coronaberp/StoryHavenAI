# Sanctum Grimoire/Masks Desktop and Ultrawide Layout Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the Grimoire (My Grimoire, lore entries) and Masks (My Masks, personas) screens a real desktop (1024–1535px) and ultrawide (≥1536px) layout instead of the generic 680px `.content-col` cap they currently share with every other screen.

**Architecture:** CSS-only change in `new_ui/css/cards.css` (the real source file — `new_ui/css/app.css` is the compiled Tailwind output and must never be hand-edited, see `CLAUDE.md`). Two new override classes widen `.content-col` at the `lg:`/`2xl:` breakpoints, and the existing `.sanctum-feed`/`.sanctum-feed-row` classes (already shared verbatim by both screens) switch from a flex column to a responsive CSS grid at `lg:`+. Two one-line JS edits add the new override class names to each screen's existing wrapper `<div>`.

**Tech Stack:** Vanilla JS (no framework), Tailwind CSS compiled via `./rebuild.sh --watch` (already running on `:3001` per project convention), plain CSS media queries in `cards.css` (Tailwind's own `lg:`/`2xl:` breakpoints are 64rem/96rem — this file uses the equivalent raw `@media (width >= 64rem)`/`@media (width >= 96rem)` syntax already established by the existing `.admin-users-content`/`.create-content` overrides).

## Global Constraints

- Never hand-edit `new_ui/css/app.css` — it is a compiled Tailwind build artifact regenerated by the watcher; all real CSS changes go in `new_ui/css/cards.css`.
- Test against the human's already-running `./rebuild.sh --watch` dev server on `:3001` — never spin up a second dev server instance for the same purpose.
- Mobile/tablet (<1024px) rendering must stay pixel-identical to today — every change is gated behind `@media (width >= 64rem)`.
- No changes to the add/edit modal flow (`openAddFlow()` in `grimoire.js`, `openAdd()` in `masks.js`) — this plan is a pure reflow, not a behavior change.
- Casts (`pantheon.js` scoped `"mine"`) and Forge (`forge.js`) are out of scope — already correct.

### Task 2: Turn `.sanctum-feed` into a responsive grid at `lg:`+

**Files:**
- Modify: `new_ui/css/cards.css:1046-1058` (existing `.sanctum-feed`/`.sanctum-feed-row` rules)

**Interfaces:**
- Consumes: nothing new from Task 1 (this CSS rule is independent of the wrapper `max-width` classes — it targets `.sanctum-feed`/`.sanctum-feed-row` directly, which both screens already render unconditionally).
- Produces: the grid layout Task 3 verifies visually. No JS-visible interface — purely a CSS reflow of markup that already exists unchanged in both `grimoire.js`'s `rowHtml()`/`bodyHtml()` and `masks.js`'s `rowHtml()`/`bodyHtml()`.

- [ ] **Step 1: Add the `lg:`+ grid override**

Open `new_ui/css/cards.css`. Find this existing block (around line 1046):

```css
.sanctum-feed {
  display: flex;
  flex-direction: column;
}
.sanctum-feed-row {
  display: flex;
  gap: 12px;
  align-items: center;
  padding: 12px 4px;
  border-bottom: 1px solid var(--color-line);
  cursor: pointer;
}
.sanctum-feed-row:last-child { border-bottom: none; }
```

Add a new media-query block immediately after the `.sanctum-feed-row:last-child` line:

```css
.sanctum-feed {
  display: flex;
  flex-direction: column;
}
.sanctum-feed-row {
  display: flex;
  gap: 12px;
  align-items: center;
  padding: 12px 4px;
  border-bottom: 1px solid var(--color-line);
  cursor: pointer;
}
.sanctum-feed-row:last-child { border-bottom: none; }
@media (width >= 64rem) {
  .sanctum-feed {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
    gap: 8px;
  }
  .sanctum-feed-row {
    border: 1px solid var(--color-line);
    border-radius: 12px;
    padding: 12px;
  }
  .sanctum-feed-row:last-child { border-bottom: 1px solid var(--color-line); }
}
```

(The `:last-child` override at the end is required — the mobile-only base rule sets `border-bottom: none` on the last row of every list, which would otherwise leave the last card in each `lg:`+ grid missing its bottom border while every other card has one.)

Expected: one match, confirming the watcher recompiled the new grid rule into the served stylesheet.
